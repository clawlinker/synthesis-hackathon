## Idea Details

### 1. Agent Delegation Registry (Score: 9)
**Why:** Agents hiring agents have no way to verify a counterpart was authorized by a real human operator to spend funds or take actions. No permissionless, agent-native delegation primitive exists.

**Stack:** ERC-8004 identity, x402 sell, agent wallet on Base, pawr.link badges, crons for expiry monitoring.

**Gap:** Base smart contract (delegation registry mapping agent 8004 ID → signed scope structs), human signing UI (Next.js + wagmi), x402-gated lookup API.

---

### 2. A2A Service Level Agreement (Score: 9)
**Why:** Financializes trust in agent-to-agent commerce by adding stake-weighted guarantees to A2A requests.

**Stack:** ERC-8004 (ID), x402 (payment), Base (staking contract), pawr.link (trust badge).

**Gap:** SLA staking contract, breach-detection logic, x402 lookup server.

---

### 3. Agent Work Notary — Proof-of-Agent-Work (Score: 8)
**Why:** Agents can claim work was done but there's no permissionless, immutable proof of *which* 8004 agent produced *which* output at *which* block.

**Stack:** ERC-8004 identity, x402 buy/sell, agent wallet on Base, pawr.link A2A, 18 live crons as data source.

**Gap:** Solidity notary contract (`keccak256(agentId, taskId, outputHash, timestamp)` → Base), cron hooks to anchor receipts, x402 pay-per-lookup, pawr.link "verified work history" badge.

---

### 4. Agent Subscription Protocol (Score: 8)
**Why:** x402 is one-shot only — no permissionless, on-chain recurring payment primitive for agents selling subscription services.

**Stack:** ERC-8004 identity, x402 buy+sell, agent wallet on Base, pawr.link A2A + widgets, 18 live crons (first real services to monetize).

**Gap:** Subscription vault contract (USDC drip-rate lock + pull withdrawal + on-chain expiry), x402 subscriber-gating middleware, pawr.link subscriber count badge.

---

### 5. Agent Task Escrow (Score: 8)
**Why:** Agent-to-agent commerce has zero on-chain accountability — an agent can take payment and ghost.

**Stack:** ERC-8004 identity (both parties), Base USDC, x402 pay-per-lookup, pawr.link A2A + widgets.

**Gap:** Escrow + attestation contract (~150 LOC on Base), output-hash release flow, lightweight dispute resolver, pawr.link "active/completed jobs" widget.

---

### 6. Agent Liveness Oracle (Score: 8)
**Why:** ACP/A2A job routing is blind — no way to verify an agent is actually running before hiring it.

**Stack:** ERC-8004, Base wallet, x402 buy+sell, 18 live crons, pawr.link A2A.

**Gap:** Heartbeat contract on Base (~80 LOC), cron job for signed heartbeats every 15 min, x402 pay-per-query server, pawr.link uptime badge.

**Note:** Shortest build time (7-9 days). Good standalone or as a module for other ideas.

---

_All ideas use our existing stack: ERC-8004 #22945, x402, Base wallet, pawr.link. The chosen direction (Agent Receipts) overlaps most with #3 (Work Notary)._
